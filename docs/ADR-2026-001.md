# ADR – Decisión de Arquitectura

**Título:** Migración a Web Components/Lit en lugar de React/Tailwind
**ID:** ADR-2026-001
**Autor:** Julian
**Fecha:** 13/03/2026
**Versión:** 1.0
**Estado:** Aprobado
**Normatividad aplicable:** ISO 9001, ADR estándar, BPMN para diagramación

## 1. 🎯 Contexto
El proyecto utiliza un stack basado en Rust/WASM + TypeScript + CSS en Astro Islands, con filosofía purista denominada diógenes.dev.style.
Se requiere definir la capa de UI sin depender de frameworks comunes (React, Tailwind, Bootstrap), buscando ligereza, control absoluto y compatibilidad con la arquitectura WASM.

## 2. 🧩 Decisión
Adoptar Web Components nativos como base de la UI, con soporte opcional de Lit para simplificar la creación de componentes, y abandonar el uso de React/Tailwind.

## 3. 📋 Justificación
- **Compatibilidad:** Web Components interoperan con cualquier stack, incluyendo Astro Islands y WASM.
- **Ligereza:** Eliminación de runtimes pesados (React) y librerías de estilos redundantes (Tailwind).
- **Control:** Se mantiene la filosofía purista, con CSS modularizado y extensiones vía Houdini/PostCSS.
- **Escalabilidad:** Lit provee sintaxis declarativa y simplificación sin sacrificar independencia.
- **Accesibilidad:** Roles ARIA y buenas prácticas pueden implementarse manualmente o con soporte ligero de Lit.

## 4. 📊 Alternativas consideradas
- **React + Tailwind:** descartado por peso, dependencia externa y filosofía contraria al enfoque purista.
- **Svelte:** atractivo por compilación a JS plano, pero se considera innecesario dado que Astro Islands + Web Components cubren el mismo objetivo.
- **Bootstrap:** descartado por estética obsoleta y sobrecarga de estilos.

## 5. ⚠️ Impacto
- **Migración:** eliminar dependencias de React/Tailwind del repositorio.
- **Capacitación:** equipo debe familiarizarse con Web Components/Lit.
- **Documentación:** actualizar manuales y guías de estilo.
- **Riesgos:** curva de aprendizaje inicial, menor comunidad que React.
- **Mitigación:** uso de Lit y documentación interna clara.

## 6. 📈 Indicadores de éxito (KPIs)
- Tamaño del bundle reducido ≥ 30%.
- Lighthouse Performance ≥ 90.
- Tiempo de carga inicial ≤ 2s.
- Consistencia visual ≥ 95% en auditorías internas.

## 7. 📜 Historial de cambios
- **v1.0 (13/03/2026):** Decisión inicial de migración a Web Components/Lit.
- **v1.1 (pendiente):** Inclusión de métricas adicionales y casos de prueba.

## 8. ✅ Estado futuro (TO-BE)
- UI basada en Web Components/Lit.
- Estilos gestionados con CSS puro + PostCSS/Houdini.
- Lógica intensiva en Rust/WASM, orquestada en Astro Islands.
- Documentación estandarizada en ADRs y BPMN.
